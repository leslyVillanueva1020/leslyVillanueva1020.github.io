History of Database Models:

The first widely used database was developed by IBM in the 1960s, called IMS (Information Management System). This was a hierarchical database, which means the data was organized in tree-like structures, but it lacked flexibility[1]. Then, in the 1970s, the relational model was developed by Edgar F. Codd. This model organized data into tables of rows and columns [2]. This is the foundation for SQL (Structured Query Language), also made by IBM, as it is a tool that interacts with relational models through commands to insert rows, create tables, run queries, etc [3]. As time went on and the internet grew, so did the data, creating new approaches to handling data, such as NoSQL databases. Databases and the models have changed to fit different needs, so understanding the models will help you understand why certain databases are chosen for specific applications.
